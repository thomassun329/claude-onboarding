CLAUDE.md vs Memory — What's the difference?

CLAUDE.md Memory
Who writes it You Claude
What it contains
Coding standards · Project structure · Build & run commands · Things Claude should always know or always do
User — who you are, your role
Feedback — corrections, preferences
Project — active decisions, volatile state
Reference — where to find things externally
Example Sarah writes in CLAUDE.md on day 1:
"This project uses React and Python. Always run npm test before committing."
Claude reads this every session. Sarah wrote it once. 		On day 3 Sarah corrects Claude:
"Stop using var — always use const."
Claude saves this to memory automatically. Sarah never touched a file.
⚠️ Claude sometimes saves to the wrong place. Audit memory every few weeks — but don't over-manage it. If you start manually writing memory files yourself, you end up with two competing sets of instructions that drift apart. You own CLAUDE.md — let Claude own memory.
